## 场地尺寸

```
  ┌────────────────────────────────────────────┐
  │                                            │  ↑
  │        ┌──────────┐                        │  │
  │        │Destination│                        │  │ 2000mm
  │        │  zone    │                        │  │
  │        │ 1400×800 │                        │  │
  │        └──────────┘  [A]        [C]        │  │
  │                         [B]                │  │
  │                     [D]                    │  │
  │                                    ┌──────┐│  │
  │                                    │Start ││  │
  │                                    │ zone ││  │
  │                                    │600×800│  │
  │                                    └──────┘│  │
  └────────────────────────────────────────────┘  ↓
  ←────────────── 3020mm ──────────────────────→
```

- **总尺寸**: 3020mm × 2000mm (≈ 3m × 2m)
- **起点区 (蓝)**: 600×800mm, 位于右下角, 距右墙 600mm, 距下墙 0mm
- **终点区 (绿)**: 1400×800mm, 位于左上角, 距上墙 300mm, 距左墙 800mm
- **4 个货架**: 分散在场地中央, 每个货架外框 140×140mm
- **场地围墙高度**: 500mm, 厚 232.5mm
- **货架内开口宽度**: 140mm (比赛规则, 叉臂 120-130mm 能插入)

## 坐标系定义

机器人 `map` 坐标系的原点选择**起点区中心**, 朝向约定:
- **+x** = 正前方 (面向场地中央)
- **+y** = 左侧
- **+θ** = 逆时针 (右手定则)

起点位姿: `(0, 0, 0)`

各货架中心坐标 (必须建图后用卷尺校验!):
```python
RACK_POSITIONS = {
    "A": (-0.75,  0.60),   # 左上象限
    "B": ( 0.15, -0.15),   # 右中
    "C": ( 0.60,  0.65),   # 右上
    "D": (-0.15, -0.15),   # 中偏左
}
DESTINATION = (-0.80, 1.10, yaw=π/2)   # 左上, 朝上
```

## QR 码规格

**6 张 PNG** 已生成在 `scripts/qr_codes/` (队伍串 `4X6M`, 换团队要改):
- `START.png`
- `RACKA_4X6M.png`, `RACKB_4X6M.png`, `RACKC_4X6M.png`, `RACKD_4X6M.png`
- `END.png`

**打印规格**:
- A4 纸, 每张 QR 约 **8×8 cm** (足够相机 0.3m 外清晰识别)
- 建议塑封, 避免卷角 / 反光
- **粘贴位置**:
  - START/END: 立在起点区/终点区的圆柱上方 (参考 PDF 第 1 页图)
  - RACK_X: 贴在货架正面中央, 离地 **120-140mm** (与相机微下倾 15° 扫描高度对齐)

## LiDAR 塔高度

上层板前方立柱托起 MS200 LiDAR 到 **120-140mm**, 使扫描平面**高于货架底部 QR 码**但**低于货架顶板**, 这样:
- LiDAR 能看到货架立柱 (定位用)
- 相机能扫到货架正面 QR

## 建图 SOP (到场必做)

1. Pi5 + 底盘通电, tmux 开 4 窗口
2. `ros2 launch our_robot slam_mapping.launch.py`
3. 另一终端 `ros2 run teleop_twist_keyboard teleop_twist_keyboard`
4. 缓速推机器人覆盖全场 2-3 圈, 看 RViz2 里地图边缘闭合
5. `bash scripts/save_map.sh` 存到 `~/maps/gamefield_map.yaml + .pgm`
6. **备份 U 盘一份**! SD 挂了不用重建

## 校验货架坐标 (建图后)

用卷尺从 START 中心量到各货架中心的 (Δx, Δy), 跟 `rack_positions.py:RACK_POSITIONS` 对比:
- 偏差 > 5cm → 手动改坐标
- 偏差 > 10cm → 重新建图

```bash
# 量完改完后测试:
ros2 launch our_robot robot_full.launch.py
# 看机器人是否准确停在每个货架前 0.3m
```
